## 部署方式选择

| 方式 | 难度 | 适用场景 | 包含服务 |
|------|------|---------|---------|
| Docker 全量 | ⭐ 低 | 生产环境推荐 | 后端 + 前端 + Nginx |
| Docker 全量 + 可选依赖 | ⭐ 低 | 需要本地 LLM 或 RSSHub | 上方基础上，额外启动 Ollama（独立 compose）/ RSSHub（--profile rsshub 或独立 compose）|
| Docker 仅后端 | ⭐ 最低 | 只需 API、已有前端部署 | 后端 |
| 传统手动 | ⭐⭐⭐ 中 | 无 Docker 环境、自定义需求 | 后端 + 自行配置前端/Nginx |

## 方式一：Docker Compose 全量部署（推荐）

### 1. 服务器环境准备

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.23.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version && docker-compose --version
```

> **⚠️ 中国大陆服务器注意事项**
>
> `get.docker.com` 和 `github.com` 在国内可能无法直接访问。如遇连接超时或被重置，替换为国内镜像源：
>
> **Docker 安装（以 Ubuntu 为例，使用阿里云镜像）：**
> ```bash
> curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
> echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null
> apt update
> apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
> ```
>
> **Docker Compose 替代方案（使用国内镜像）：**
> ```bash
> sudo curl -L "https://mirror.ghproxy.com/https://github.com/docker/compose/releases/download/v2.23.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
> sudo chmod +x /usr/local/bin/docker-compose
> ```
>
> 其他发行版请自行替换对应镜像源，或搜索"国内 Docker 安装镜像"。

### 2. 获取项目代码

```bash
git clone <你的仓库地址> ai_news_bot
cd ai_news_bot
```

### 3. 创建必要目录

```bash
mkdir -p storage logs ssl
# storage/  ── 持久化数据（SQLite 数据库、ChromaDB 向量库、Agent 记忆）
# logs/     ── 运行日志
# ssl/      ── SSL 证书（用于 Nginx HTTPS）
```

### 4. 配置环境变量

```bash
# 注意：Docker Compose 读取的是 ./app/.env 路径
cp .env.example app/.env
nano app/.env
```

**核心必填项：**
```bash
SECRET_KEY=your-secret-key-at-least-32-characters-long
WEB_PANEL_PASSWORD=your-strong-password

# 如使用 Ollama 本地模型，取消注释：
# OLLAMA_BASE_URL=http://ollama:11434
```

> **LLM API Key 无需写在 .env 中**，部署完成后通过 Web 面板的"我的 LLM"页面配置，密钥加密后存入数据库，即时生效。

完整配置项说明见下方[环境变量完整说明](#环境变量完整说明)。

### 5. 配置 SSL 证书（有域名时）

```bash
# 方式 A：Let's Encrypt（推荐）
sudo certbot certonly --nginx -d your-domain.com
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem ssl/
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem ssl/
sudo chmod 644 ssl/*.pem

# 方式 B：使用已有证书
# 将证书文件复制到 ssl/ 目录，命名为 fullchain.pem 和 privkey.pem
```

### 6. 配置 Nginx 域名

编辑 `nginx/conf.d/ai-news-bot.conf`，将 `your-domain.com` 替换为你的实际域名。

### 7. 启动所有服务

```bash
# 构建并启动所有服务
docker-compose up -d --build

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 8. RSSHub 启动说明

RSSHub 提供 300+ 数据源扩展。**默认不启动**，用户可按需选择以下任一方式启动：

#### 方式一：Compose Profiles（推荐）

```bash
# 与主服务一起启动（含 RSSHub）
docker-compose --profile rsshub up -d

# 仅启动 RSSHub（主服务已运行时）
docker-compose --profile rsshub up -d rsshub
```

#### 方式二：独立文件

```bash
docker-compose -f docker-compose.rsshub.yml up -d
```

#### 自动启动逻辑

项目内置 **RSSHub Manager**，应用启动时会自动检测 RSSHub 是否已在运行：

```
启动 → 检测 rsshub_enabled 配置
     → HTTP 健康检查 → RSSHub 已运行？→ 是 → 跳过 Docker 检测，标记 RUNNING
                      → 否 → 检测 Docker 环境 → rsshub_auto_start? → 启动 RSSHub 容器
```

- **本地开发**（uvicorn 直接启动）：如 Docker 可用且 `rsshub_auto_start=true`，会自动拉起 RSSHub 容器
- **Docker 部署**：推荐通过 `--profile rsshub` 启动，Manager 会检测到 RSSHub 已运行并自动同步状态

> ⚠️ 前提条件：
> - 服务器已安装 Docker 且 Docker daemon 正在运行
> - `.env` 中 `rsshub_enabled=true`（默认）且 `rsshub_auto_start=true`（默认）
>
> 如需禁用 RSSHub，在 `.env` 中设置：
> ```bash
> RSSHUB_ENABLED=false
> ```

验证 RSSHub 是否正常运行：

```bash
# 检查 RSSHub 容器状态
docker ps | grep rsshub

# 直接访问 RSSHub 服务
curl http://localhost:1200/

# 通过 Web 面板查看 RSSHub 状态 → "RSSHub帮助" 页面
```

### 9. （可选）启动 Ollama 本地 LLM

Ollama 是可选依赖，已从主 `docker-compose.yml` 中分离，需通过独立文件启动：

```bash
# 启动 Ollama 容器
docker-compose -f docker-compose.ollama.yml up -d

# 拉取并运行模型（示例：llama3）
docker exec -it ai-news-bot-ollama ollama pull llama3

# 在 .env 中设置 Ollama 地址（Docker 网络内部使用容器名访问）
# 编辑 app/.env，添加或取消注释：
# OLLAMA_BASE_URL=http://ollama:11434

# 重启 app 容器使配置生效
docker-compose restart app
```

> ⚠️ 注意：Ollama 依赖主 compose 文件的 `ai-news-network` 网络，所以必须先启动主服务（`docker-compose up -d`），再单独启动 Ollama。

### 10. 访问服务

| 服务 | 地址 | 说明 |
|------|------|------|
| Web 面板 | `https://your-domain.com/` 或 `http://服务器IP` | 前端管理界面 |
| API 文档 | `https://your-domain.com/docs` | FastAPI Swagger 文档 |
| 健康检查 | `https://your-domain.com/health` | 服务状态监控 |
| RSSHub | `http://localhost:1200` | RSSHub 服务（内部使用，通过 --profile rsshub 启动）|

### 启动后的初始化配置

首次登录 Web 面板后，建议按以下顺序进行初始设置：

1. **配置 LLM** → "我的 LLM" → 添加 Provider（智谱/硅基流动/OpenRouter 等）
2. **配置 Embedding** → "向量配置" → 选择 Embedding 模型
3. **配置推送** → "Webhook 配置" → 添加企业微信/Telegram 等
4. **检查 RSS 源** → "RSS 源管理" → 确认内置源已激活
5. **手动触发一次采集** → "定时任务" → 点击 `fetch_ai_news` 的"触发"按钮

## 方式三：传统手动部署

### 1. 服务器环境准备（Ubuntu/Debian）

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Python
sudo apt install -y python3.11 python3.11-venv python3-pip

# 安装 Nginx
sudo apt install -y nginx

# 安装 Git
sudo apt install -y git

# 安装 SSL 证书工具（有域名时）
sudo apt install -y certbot python3-certbot-nginx

# 安装 Node.js 18+（需要部署前端时）
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
```

### 2. 获取项目代码

```bash
cd /var/www/
sudo git clone <你的仓库地址> ai_news_bot
sudo chown -R $USER:$USER ai_news_bot
cd ai_news_bot
```

### 3. 配置后端

```bash
# 创建虚拟环境
python3.11 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 创建必要目录
mkdir -p storage logs

# 配置环境变量
cp .env.example .env
nano .env
# 至少配置 SECRET_KEY 和 WEB_PANEL_PASSWORD
```

### 4. 初始化数据库

```bash
# 首次启动会自动创建数据库和表结构
# 也可以手动运行迁移（如有需要）
source venv/bin/activate
python -c "import asyncio; from app.database import init_database; asyncio.run(init_database())"
```

### 5. 配置 Nginx

```bash
sudo nano /etc/nginx/sites-available/ai-news-bot
```

写入以下配置（适用于纯后端部署，前端另行部署）：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # API 代理到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 健康检查
    location /health {
        proxy_pass http://127.0.0.1:8000/health;
        proxy_set_header Host $host;
    }

    # 前端静态文件（如需同机部署前端，编译后将 dist 复制到以下路径）
    # location / {
    #     root /var/www/ai_news_bot/web-panel/dist;
    #     index index.html;
    #     try_files $uri $uri/ /index.html;
    # }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/ai-news-bot /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx
```

### 6. 配置 SSL 证书（有域名时）

```bash
sudo certbot --nginx -d your-domain.com
```

### 7. 部署前端（如需同机部署）

```bash
cd /var/www/ai_news_bot/web-panel
npm install
npm run build
# 编译产物在 web-panel/dist/ 目录
# 将 dist 目录配置到 Nginx 的静态文件服务
```

### 8. 创建 Systemd 服务

```bash
sudo nano /etc/systemd/system/ai-news-bot.service
```

```ini
[Unit]
Description=AI News Bot Backend
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/ai_news_bot
Environment="PATH=/var/www/ai_news_bot/venv/bin"
ExecStart=/var/www/ai_news_bot/venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start ai-news-bot
sudo systemctl enable ai-news-bot
sudo systemctl status ai-news-bot
```

### 9. 查看日志

```bash
# 查看服务日志
sudo journalctl -u ai-news-bot -n 100 -f

# 查看应用日志文件
tail -f logs/app.log
```

docker-compose build --no-cache app  # 重新构建后端
docker-compose up -d                 # 重启所有服务

# 手动方式
git pull
source venv/bin/activate
pip install -r requirements.txt

### 备份数据

```bash
# SQLite 数据库 + ChromaDB 向量库
tar -czf backup_$(date +%Y%m%d).tar.gz storage/
```

### 常用 Docker 命令

```bash
# 查看所有容器状态
docker-compose ps

# 重启特定服务
docker-compose restart app

# 停止所有服务
docker-compose down

# 停止并删除数据卷（⚠️ 会清除数据库）
# docker-compose down -v

# 进入容器
docker-compose exec app bash
docker-compose exec web sh

# 查看资源占用
docker stats
```

## 常见问题

### Q1: 启动失败，端口被占用

```bash
# 检查端口占用
sudo lsof -i:8000
sudo lsof -i:80
sudo lsof -i:443

# 如端口被占用，停止占用进程或修改配置
```

### Q2: SQLite 数据库锁错误

```bash
# 通常无需处理，系统已配置 WAL 模式和 30 秒超时
# 如持续出现，检查是否有多个进程同时写入
```

### Q3: LLM 调用返回 429 限流

- 通过 Web 面板"我的 LLM"降低该 Provider 的"最大并发数"
- 或添加多个 Provider 做负载均衡

### Q4: ChromaDB 向量库不可用

```bash
# 检查 ChromaDB 数据目录权限
ls -la storage/chromadb/

# 系统会自动进行健康检查（每 30 秒），并在恢复正常后自动启用
# 如长时间不可用，可尝试重建：
docker-compose restart app
```

### Q5: RSSHub 连接失败

```bash
# 检查 RSSHub 容器状态
docker ps | grep rsshub

# 检查 RSSHub 日志
docker logs rsshub

# 确认 RSSHub 端口 1200 可访问

# 如未启动，通过以下方式启动：
docker-compose --profile rsshub up -d
# 或
docker-compose -f docker-compose.rsshub.yml up -d
```

### Q6: Web 面板无法登录

- 确认 `.env` 中 `WEB_PANEL_PASSWORD` 已正确配置
- 首次启动会自动创建管理员用户，修改密码后需重启
- 查看后端日志确认是否有认证错误

### Q7: 企业微信消息推送失败

- 确认 Webhook URL 和密钥配置正确（Web 面板 → Webhook 配置）
- 检查推送日志确认错误原因
- 查看后端日志中详细的错误信息

### Q8: Docker 部署时 storage 目录权限

```bash
# 如遇到权限问题，手动设置权限
sudo chown -R 1000:1000 storage/
```

### Q9: 定时任务不执行

```bash
# 检查调度器状态（通过 API）
curl http://localhost:8000/api/scheduler/jobs

# 检查任务执行历史
curl http://localhost:8000/api/task-history

# 确认 timezone 配置正确（默认 Asia/Shanghai）
```
