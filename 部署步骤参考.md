## 第一步：服务器环境准备

### 1.1 更新系统

## 第三步：配置Nginx反向代理

### 3.1 创建Nginx配置

```bash
sudo nano /etc/nginx/sites-available/ai_news_bot
```

写入以下内容：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名

    location / {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 3.2 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/ai_news_bot /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启Nginx
sudo systemctl restart nginx
```

## 第六步：创建Systemd服务

### 6.1 创建服务文件

```bash
sudo nano /etc/systemd/system/ai-news-bot.service
```

写入以下内容：

```ini
[Unit]
Description=AI News Bot - AI资讯与GitHub热门项目自动收录工具
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/home/ai_news_bot_proj/ai_news_bot
Environment="PATH=/home/ai_news_bot_proj/ai_news_bot/venv/bin"
ExecStart=/home/ai_news_bot_proj/ai_news_bot/venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8001
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 6.2 启动服务

```bash
# 重新加载systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start ai-news-bot

# 设置开机自启
sudo systemctl enable ai-news-bot

# 查看服务状态
sudo systemctl status ai-news-bot

# 查看详细日志
sudo journalctl -u ai-news-bot -f
```

## 第七步：配置企业微信自建应用

### 7.1 创建自建应用

1. 登录 **企业微信管理后台** → https://work.weixin.qq.com/
2. 进入 **工作台** → **自建应用**
3. 点击 **创建应用**
4. 填写：
   - 应用名称：`AI资讯Bot`
   - 应用描述：`AI资讯与GitHub热门项目推送`
5. 上传应用Logo（可选）
6. 点击 **创建**
7. 在应用页面 → **可见范围** → 设置可使用的人员

### 7.2 配置API接收消息

1. 进入创建的应用 → **API接收消息** → **设置**
2. 配置：
   - **URL**：`https://your-domain.com/webhook/wecom`
   - **Token**：填写在 `.env` 中生成的 `WECOM_TOKEN`
   - **EncodingAESKey**：填写在 `.env` 中生成的 `WECOM_AES_KEY`
3. 点击 **保存**

### 7.3 获取应用凭证

在应用页面找到：
- **AgentId**（应用ID）→ 填入 `WECOM_AGENT_ID`
- **Secret**（应用密钥）→ 填入 `WECOM_AGENT_SECRET`

在企业管理后台 → **我的企业** → **企业信息** 找到：
- **CorpID**（企业ID）→ 填入 `WECOM_CORP_ID`

## Docker部署方式

> 如果你更倾向于使用Docker部署，可以选择以下方式。

---

### 方式一：Docker部署（推荐）

#### 3.1 安装Docker

```bash
# 安装Docker
curl -fsSL https://get.docker.com | sh

# 启动Docker
sudo systemctl start docker
sudo systemctl enable docker

# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.23.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

#### 3.2 配置SSL证书目录

```bash
# 创建SSL证书目录
mkdir -p ssl

# 使用certbot获取证书（推荐）
certbot certonly --nginx -d your-domain.com

# 复制证书到项目目录
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem ssl/
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem ssl/
sudo chmod 644 ssl/*.pem
```

#### 3.3 配置Nginx

编辑 `nginx/conf.d/ai-news-bot.conf`，将 `your-domain.com` 替换为你的域名：

```bash
nano nginx/conf.d/ai-news-bot.conf
```

#### 3.4 配置环境变量

```bash
# 复制配置模板
cp .env.example app/.env
nano app/.env
```

#### 3.5 启动Docker容器

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

#### 3.6 验证

```bash
# 测试API
curl https://your-domain.com/

# 测试健康检查
curl https://your-domain.com/health
```

---

### 方式二：纯Docker（不带Nginx）

如果你只想用Docker运行应用，不需要Nginx，可以：

```bash
# 编辑docker-compose.yml，注释掉nginx服务
# 或者直接运行

# 构建镜像
docker build -t ai-news-bot -f docker/Dockerfile .

# 运行容器
docker run -d \
  --name ai-news-bot \
  -p 8001:8000 \
  -v ./storage:/app/storage \
  -e DATABASE_URL=sqlite+aiosqlite:///app/storage/database.db \
  -e WECOM_CORP_ID=your-corp-id \
  -e WECOM_AGENT_ID=your-agent-id \
  -e WECOM_AGENT_SECRET=your-secret \
  -e WECOM_TOKEN=your-token \
  -e WECOM_AES_KEY=your-aes-key \
  -e OPENAI_API_KEY=sk-xxx \
  ai-news-bot
```

然后配置Nginx反向代理（参考方式一中的Nginx配置）。
